GAP 1: "Influenced by"

We noticed that the object under “influenced by” (P737) is deprecated. On wikidata, a triple statement can be deprecated for a variety, such as being no longer true or having an unreliable source, such as this case. As a result, it doesn’t show up when you query it. To be sure, we ran a query, comparing Nabokov’s information with that of a similar writer, Joseph Brodsky (Q862), to see which objects they have under the property “influenced by” (P737). We used the keyword OPTIONAL so the results would show up even if the triple pattern doesn’t exist.

PROMPTING TECHNIQUES

SELECT ?subject ?subjectLabel ?influencedBy ?influencedByLabel 
WHERE
 { VALUES ?subject  { wd:Q36591 wd:Q862 }  
 OPTIONAL { ?subject wdt:P737 ?influencedBy . }
 SERVICE wikibase:label { bd:serviceParam wikibase:language "en". } }
ORDER BY ?subjectLabel
LIMIT 50

While ChatGPT’s answer was concise and identified his key influences, it was also factually incorrect regarding Fitzgerald, whom Nabokov disliked. We preferred Gemini’s more detailed response, which also provided information about Nabokov’s contrarian opinions. We decided to focus on Pushkin, Gogol and Tolstoy, his key influences. Sometimes, an author’s influence on another author is expressed through other properties, such as “student of” (P1066), so we decided to run another query, to see which items are linked to the objects Alexander Pushkin (Q7200), Lev Tolstoy (Q43718) and Nikolai Gogol (Q7243) under any property. We added the P31 and Q5 to make sure that the results would be humans and not Wikipedia links or other items.
RESULT CODE 
SELECT DISTINCT ?item ?itemLabel 
WHERE {   ?item wdt:P31 wd:Q5 .
  { ?item ?property wd:Q7200 .}
  UNION  
  { ?item ?property wd:Q43718 .}
  UNION
  { ?item ?property wd:Q7243 .}
  SERVICE wikibase:label { bd:serviceParam wikibase:language "en". }
}
ORDER BY DESC(?item)
LIMIT 100
